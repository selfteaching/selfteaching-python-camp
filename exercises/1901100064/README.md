2019.9.17 我的自学总结
一、关于“自学”
自学不是独自学习，我认为，应该是自主学习――能够主动调用一切资源为自己的学习服务。
千万不要独自学习。
学习编程也是这样，由于是网络学习，我感觉学员之间的交流普遍偏少。如果我们大部分时
间里都在独自学习，那就失去了共同学习的意义。

二、关于阅读参考资料
在编程学习中，我们确实应该具备阅读英文文档的能力。但在我们初学编程时，应先从阅读
容易理解的中文文档开始。否则，在起步阶段会花费太多的时间，事倍而功半。
如果对中文文档有疑问，可以再去调阅英文文档。
在成为熟手之后，可以去研究英文文档。

三、关于软件工程
我发现，软件工程的基本思想与土木工程、环境工程等工程学习的基本思想是一致的。遵循
一系列的语法、标准、规范，在标准规定的范围内工作。工作起来的工作过程也是相似的。
例如，对于爬虫而言，如果系统去学，需要学习与之相关的很多知识点，不是容易的事情。
有非常多的相互关联的细小的知识点。

四、关于“创作”
我发现，在编程领域内进行创作也不是容易的事。就像在传统工程领域，想要发明一种新工艺
、新方法是很难的。在绝大多数时间里，我们其实都是在学习前辈们的代码，或者在前人积累
的代码的基础上进行修改完善。
只有当我们也成为了熟手、高手之后，并且在进入到一个全新的领域之后，你才有机会进行创作。
