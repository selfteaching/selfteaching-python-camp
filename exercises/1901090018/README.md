收获总结：掌握Github上的重点操作，
1，建立Repository（仓库），Repository就是一个项目的汇总，里面包含各样文件。
2，建立Branch（分支），目的就是不影响主分支Branch Master，在新的Branch 里添加，修改，删除数据。
3，Commit，分支上有修改的时候就要Commit才能确认修改。
4，Pull Request和Merge，这个Pull Requset可以是提交到自己的Branch里，也可以是提交到别人的Master Branch。就是提交合并请求，通过对比前后修改内容，Master Branch的主人如果觉得这个修改可以操作就Merge，这时候新的修改就产生了。
5，添加issue，类似添加各类标签，方便自己和别人归类查阅。

学习心得：因为在报班之前看过官方说明，也自己操作过一次这几个步骤。第二次看官方说明的时候，就是温故知新，更加明白这里所说的，所以一路操作都比较顺利。
