# Learn Python

挫折

所谓万事开头难，每做一件事前我知道起步阶段是困难的。然而开始第一天任务时，我没有想到会如此之难，居然花了我三天时间，去纠结一个本已完成而我不知道已经完成
的问题。

根据视频的提示，我在自己的库上操作了一遍，所有的过程基本顺利完成，除了有点不适应英文。之后我尝试交作业，按视频步骤操作一遍，发现不一样了，没有了最后一步“Merge pull request”.
是的，没有了“Merge pull request”――到后来，我咨询了群里其他同学的意见，才明白其实我是做到了，只是我不知道而已。在自己的库里修改操作，是可以自己同意合并；当你修改了别人的库的内容时，
当然需要别人的同意才能合并。

看书千百遍，不如实际操作一遍。我用了三天时间，领悟了一个道理：问题并没有想象中的难，问题出现了一定要多渠道去寻找答案，不要困死在一棵树上。在经过第一天任务之后，我有点开窍，在后面课程任务上，
就算遇到语法问题和意外，我都会很耐心地阅读错误笔记，因为大多数错误都写在了日志了，先自己尝试着理解和解决，比如知乎百度goolge,或者在群里提问，基本上都能解决问题。

成长

说句实话，每天任务里的参考阅读资料，我都没有看懂。但人是可以学习自己不懂的东西的，所以呢，我就按视频直接操作了几遍，一遍不够，要多操作修改几遍，多百度多google，之后的任务我都用这个方法来练习，
来熟练步骤，有些东西的确需要操作过才发现问题，比如语句缩进、比如大小写、不如内建函数，没有操作过的人是不用注意到这些小细节，然而就因为这些小细节的出现，造成程序的无法运行。

从第二天到第十三天任务，基本上我都能仅靠自己来完成，虽然有些任务不易懂耗费些时间，但是结果还是好的。以前会认为完成了一件事情一项任务很有成就感，到现在完成了多项任务习惯了这种节奏之后，坦然接受。

毒品会让人上瘾，游戏也能让人上瘾，而成就感亦是。虽然完成了十三天的任务，我也只认为自己只是掌握了自学的一些小皮毛，以后更是需要深入锻炼。

感悟

人至践则无敌。唯有实践是验证想法或一切事情的唯一标准，做人做事都如此。用行动来修改和更新自己的理论――我现在看《自学是门手艺》这本书，在前几次我完成搞不懂，一头雾水到现在我能理解句子的意思和方法。
整个人犹如开了窍似的，这种喜悦感一生难忘。更让我相信，有些事情想不通，就直接做一遍，后面自热而然就全部理解了。


最后，我要说：刚开始看到课程时就被价格吓倒了，2016元！一个字贵！但是，标榜着自学能力和编程入门基础，我就硬着头皮交钱了（人不冲动做不了决定）。到如今我决定那2000多块钱，值！光自学能力这种颠覆以往
认知的技能和思维，在现代社会没有人教导，也没有具体的教程学习。当你完完整整靠自己去做到之后，你会自信以后没有什么难事了。
